# Planning Poker — Ideas & Improvements

## UX / UI

- [ ] **Анимация переворота карт** — при reveal карты игроков переворачиваются с 3D-анимацией (CSS transform rotateY)
- [ ] **Таймер на голосование** — обратный отсчёт (настраиваемый: 30с/1м/2м), по истечении автоматический reveal
- [ ] **Звуковые уведомления** — звук при входе игрока, при reveal, при переходе к следующему вопросу
- [ ] **Конфетти при консенсусе** — если все проголосовали одинаково, запускать анимацию конфетти
- [ ] **Drag & drop вопросов** — при создании комнаты перетаскивание вопросов для изменения порядка
- [ ] **Тёмная/светлая тема** — переключатель темы, сохранение в localStorage
- [ ] **Аватарки** — выбор аватарки/эмодзи при входе в комнату, отображение на карточке игрока
- [ ] **Индикатор "печатает"** — показывать что участник выбирает карту (hovering)
- [ ] **Мобильная оптимизация** — bottom sheet для карт голосования, swipe для навигации

## Функциональность

- [ ] **Пароль на комнату** — опциональный пароль при создании, запрос при входе
- [ ] **Импорт из Jira/CSV** — загрузка задач из файла или API Jira/YouTrack
- [ ] **Экспорт в Jira** — отправка оценок обратно в Jira через API
- [ ] **Комментарии к карточкам** — чат/заметки к каждому вопросу во время обсуждения
- [ ] **Голосование "кофе-брейк"** — специальная карта "кофе" для паузы
- [ ] **Статистика расхождений** — подсветка самой высокой и самой низкой оценки, показ разброса
- [ ] **Повторный просмотр вопросов** — возврат к предыдущему вопросу для пересмотра оценки
- [ ] **Шаблоны шкал** — пользовательские шкалы (ввод своих значений)
- [ ] **Роль "модератор"** — отдельная роль без голоса, но с правом управления (не владелец)
- [ ] **Кик игроков** — владелец может удалить участника из комнаты
- [ ] **Присоединение как наблюдатель** — вход в комнату без голоса (зритель)

## Техническое

- [ ] **Персистентность** — сохранение комнат в SQLite/LiteDB чтобы переживали рестарт
- [ ] **Reconnect по ID** — при обрыве соединения игрок восстанавливает сессию (сейчас теряется)
- [ ] **Rate limiting** — защита от спама создания комнат
- [ ] **TTL для комнат** — автоудаление неактивных комнат через N часов
- [ ] **Health check endpoint** — `/health` для мониторинга через HA/Cloudflare
- [ ] **Логирование** — структурированные логи (Serilog) для отладки
- [ ] **Метрики** — количество активных комнат, игроков, среднее время голосования
- [ ] **PWA** — manifest.json + service worker для "установки" на телефон
- [ ] **QR-код** — генерация QR-кода со ссылкой на комнату для быстрого входа с телефона

## Интеграции

- [ ] **Telegram бот** — уведомления о начале сессии, результатах
- [ ] **Slack integration** — slash-команда `/poker create` для создания комнаты из Slack
- [ ] **Home Assistant automation** — уведомление в HA при завершении сессии (через HA API)
- [ ] **GitHub Issues** — импорт issues как карточек для оценки
- [ ] **Webhook на завершение** — POST-запрос с результатами на произвольный URL
